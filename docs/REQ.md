# 요구사항 명세 (REQ)

## 1. 서비스 개요

프롬프트를 템플릿(설명+변수폼+완성 프롬프트) 형태로 저장·공유하고,
사용자는 변수만 채우면 즉시 복사/전송 가능한 완성 프롬프트를 얻는 서비스.

## 2. 핵심 기능 요구사항

### 2.1 비로그인 사용자
- REQ-001: 메인 페이지에서 주간 인기/랜덤/신규/태그 프롬프트를 카드 형태로 탐색 가능
- REQ-002: 탐색 페이지에서 태그/정렬/검색(제목, 태그)으로 프롬프트 필터링
- REQ-003: 상세 페이지에서 설명 + Prompt Builder(변수 폼) + 완성 프롬프트 미리보기 + 복사 가능
- REQ-004: "Chat UI로 사용" 버튼으로 클립보드 복사 + 새 탭 Chat UI 오픈

### 2.2 로그인 사용자
- REQ-010: Supabase Auth 기반 소셜/이메일 로그인
- REQ-011: 프롬프트 즐겨찾기(북마크) 저장/해제
- REQ-012: 내 라이브러리에서 즐겨찾기 목록 조회
- REQ-013: 프롬프트 작성/편집/발행 (초안→발행)
- REQ-014: 프롬프트 버전 관리 (새 버전 생성)
- REQ-015: 좋아요 (선택)

### 2.3 관리자
- REQ-020: 프롬프트 신고/차단/비공개 처리 (최소 스켈레톤)

## 3. 템플릿 규칙

- REQ-030: 본문에 `{{var}}` 플레이스홀더 사용
- REQ-031: 변수 스키마: key/label/type/required/placeholder/helpText/defaultValue/options/validation
- REQ-032: 필수 변수 미입력 시 Generate 비활성
- REQ-033: 선택 변수 미입력 시 빈 문자열 치환 (MVP)

## 4. 인기/랜덤 알고리즘

- REQ-040: 주간 score = bookmarks*3 + likes*2 + generate*1 (최근 7일)
- REQ-041: 동일 사용자 generate 하루 n회 제한 (서버 단)
- REQ-042: 비로그인 레이트리밋 (간단)

## 5. 비기능 요구사항

- REQ-050: RLS 미사용 (애플리케이션 레이어에서 권한 강제)
- REQ-051: FK 제약조건 미사용
- REQ-052: soft-FK 검사 미사용
- REQ-053: JWT 검증은 Supabase JWKS 기반 (단일 인증원)
- REQ-054: API 에러 응답 표준화 (HTTP + 내부 code + message)
